# Daily Aviation Brief

按固定模板输出一份中文简报，严格限制为最近24小时信息。

## 语言规则

- 使用简体中文输出全部标题、摘要、判断和说明。
- 保留机构名或雇主名原文可接受，但解释文字必须是中文。
- 避免直译腔，保持专业、简洁、可执行。

## 执行流程

1. 采集航空新闻与低时长飞行员相关岗位信息。
2. 校验发布时间或职位发布时间，只保留运行时刻向前24小时内的数据。
3. 若仅有日期无具体时间，仅在该日期等于用户本地“今天”时保留。
4. 按标题、来源、URL去重，重复项保留最新一条。
5. 按行业影响力（新闻）与可投递性（岗位）排序。
6. 严格按 `references/output-template.md` 的标题、列名和顺序输出。

## 新鲜度硬约束

- 计算窗口：`current_time - 24h` 到 `current_time`。
- 必须可验证时间戳；无法验证则丢弃。
- 不复用历史缓存内容。
- 每条记录必须附源链接。

## 新闻筛选

- 优先：安全通告、监管政策、航司运行、机队与培训、劳动力与市场信号。
- 排除：纯观点、无来源、明显二手搬运且无原始链接内容。

## 岗位筛选

- 仅收录低时长相关岗位：CFI、航测、跳伞、拖旗、管线巡检、SIC入门、区域航司通道、cadet项目等。
- 排除明显高门槛岗位（如ATP硬门槛且无过渡项目说明）。
- 同一雇主多条近似岗位按一个机会计入“投递建议”判断。

## 低时长分层判定

- 0-250小时：入门级。
- 250-500小时：核心低时长段。
- 500-1000小时：上沿低时长段。
- 1000+小时：默认不算低时长；仅在明确写有桥接/培养通道时纳入观察。

在岗位表的“低时长匹配度”中明确写出分层与可达性判断，避免乐观推断。
## 航空要闻与招聘（至少10条，过去24小时）

本节汇总过去24小时内的航空行业要闻与低时长飞行员岗位机会。

**数量硬约束**：每次输出必须至少10条。不足时必须用占位条目补足，并明确标注"占位：未找到足够新鲜来源/信息不足"。

**新鲜度硬约束**：
- 每条必须附发布时间戳或来源页面时间戳；无法确认则标注"时间戳不明"并降低优先级

**Sources 硬约束**：
- 每条必须至少包含2个完整 URL 链接（允许仅1个但必须写明原因）
- Sources 必须是可点击的网站链接或 YouTube 链接

每条固定格式如下：

```
# 标题（公司/事件/岗位）

- 类型：要闻 / 招聘
- 地点/地区：<具体地区，如中国/美国/澳大利亚>
- 发布时间：<YYYY-MM-DD HH:MM UTC 或本地时区>
- 摘要（2-3句，包含对低小时飞行员的意义）：<核心内容>
- 今日是否值得投简历：值得 / 谨慎 / 不建议（给1句理由）
- Sources:
  - [源1标题](https://example.com)
  - [源2标题](https://example.com)
  - YouTube（可选）：[视频标题](https://youtube.com/watch?v=xxxxx)
```

**示例**：

```
# XXX航司CFI招聘计划启动

- 类型：招聘
- 地点/地区：美国德州
- 发布时间：2026-02-06 08:30 UTC
- 摘要（2-3句）：知名区域航司启动春季CFI招聘，需要250+小时。这是低时长飞行员获取教学经验与构建简历的直接路径。该航司历史上对低时长CFI转正通道清晰。
- 今日是否值得投简历：值得（符合低时长岗位定义，招聘期活跃）
- Sources:
  - [XXX航司官网招聘页](https://example.com/careers)
  - [行业论坛讨论](https://example.com/forum)
```
## 今日投递建议规则

- 报告第一部分必须为 `## 今日投递建议`。
- 结论仅允许：`值得投递` / `谨慎投递` / `不建议投递`。
- 需给出1-2条简短理由，依据岗位数量、时长匹配、地域匹配、来源可信度与新鲜度。

判定标准：

- `值得投递`：至少1个0-1000小时段强匹配岗位，要求清晰且可直接申请。
- `谨慎投递`：仅边缘匹配、时长要求不清或地域限制较强。
- `不建议投递`：过去24小时无合格低时长岗位。

若存在签证/基地限制等明显地域不匹配，结论下调一级。

## 空结果占位

当新闻或岗位无合格条目时，在对应表格输出单行：
`过去24小时内无符合条件的信息。`

## 结束行

报告末尾固定输出：
`Data window: <start ISO 8601> to <end ISO 8601>`
